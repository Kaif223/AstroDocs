## üìù Using the CMS Interface

### Dashboard Overview
- **Documentation**: Main content pages
- **Guides**: Step-by-step tutorials  
- **Reference**: Technical documentation
- **Media**: Upload and manage images

### Creating New Content

#### Option 1: New Documentation Page
1. Click **"Documentation"** in the left sidebar
2. Click **"New Documentation"** button
3. Fill in the form:
   - **Title**: Your page title
   - **Description**: Brief summary (optional)
   - **Body**: Your content using the rich text editor

#### Option 2: New Guide
1. Click **"Guides"** in the left sidebar
2. Click **"New Guide"** button
3. Use the same form interface

### Rich Text Editor Features

#### Text Formatting
- **Bold**: Select text and click **B**
- **Italic**: Select text and click **I**
- **Headers**: Use the dropdown to select H1, H2, H3, etc.
- **Lists**: Click bullet or numbered list buttons

#### Adding Links
1. Select the text you want to link
2. Click the link button (üîó)
3. Enter the URL
4. Click "Apply"

#### Adding Images
1. Click the image button (üñºÔ∏è)
2. Choose to upload a new image or select existing
3. Add alt text for accessibility
4. Click "Insert"

#### Code Blocks
1. Click the code button (</>)
2. Select the programming language
3. Paste your code
4. Click "Apply"

### Saving Your Work

#### Save as Draft
- Click **"Save"** to save without publishing
- Content won't appear on the live site yet
- You can continue editing later

#### Publish Content
- Click **"Publish"** to make content live
- Changes will appear on the website after deployment
- Usually takes 1-2 minutes to go live

## üîÑ Content Workflow

### Editorial Workflow (Recommended for Teams)
If enabled, the CMS supports an approval process:

1. **Draft**: Content creator writes content
2. **In Review**: Submit for review
3. **Ready**: Reviewer approves
4. **Published**: Content goes live

### Direct Publishing
- Content goes live immediately upon publish
- Good for trusted content creators
- Faster but no review process

## üÜò Troubleshooting

### Can't Login?
- Make sure you have GitHub account access
- Contact repository administrator for permissions
- Try clearing browser cache

### Content Not Appearing?
- Check if you clicked "Publish" not just "Save"
- Wait 1-2 minutes for deployment
- Check GitHub Actions for deployment status

### Rich Editor Issues?
- Try refreshing the page
- Clear browser cache
- Use a different browser (Chrome/Firefox recommended)

### Lost Content?
- Check "Drafts" section
- Content is auto-saved frequently
- Contact administrator if content is truly lost
